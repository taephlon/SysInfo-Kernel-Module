# sysinfo_plus — System Information Kernel Module

A Linux kernel module that creates `/proc/sysinfo_plus` — a virtual file that
exposes live system information: CPU count, RAM usage, kernel version, uptime,
and load averages, all read directly from kernel data structures without
touching disk.

```
cat /proc/sysinfo_plus

==============================================
  /proc/sysinfo_plus — kernel system info
==============================================

CPU cores       : 8 online  (8 possible)
Kernel version  : 6.5.0-45-generic
Total RAM       : 15856 MiB
Free RAM        : 9204 MiB
Used RAM        : 6652 MiB  (41%)
Buffer cache    : 312 MiB

Uptime          : 2 days, 4 hours, 17 min, 38 sec
Load average    : 0.42  0.38  0.31  (1 min / 5 min / 15 min)
```

## Quick Start

**Install kernel headers:**

```bash
# Debian / Ubuntu
sudo apt install linux-headers-$(uname -r) build-essential

# Fedora / RHEL
sudo dnf install kernel-devel gcc make
```

**Build, load, read, unload:**

```bash
make
sudo insmod sysinfo_plus.ko
cat /proc/sysinfo_plus
sudo rmmod sysinfo_plus
```

**Verify the module is loaded:**

```bash
lsmod | grep sysinfo_plus
dmesg | tail -5
```

## Kernel APIs Used

Each field in the output is backed by a specific kernel API. The table below
maps field → API → header.

| Field | Kernel API / symbol | Header |
|---|---|---|
| CPU count | `num_online_cpus()`, `num_possible_cpus()` | `<linux/cpumask.h>` |
| Kernel version | `utsname()->release` | `<linux/utsname.h>` |
| Total / Free RAM | `si_meminfo()` → `struct sysinfo` | `<linux/mm.h>` |
| Buffer cache | `si.bufferram` (from `si_meminfo`) | `<linux/mm.h>` |
| Uptime | `jiffies`, `INITIAL_JIFFIES`, `HZ` | `<linux/jiffies.h>` |
| Load average | `get_avenrun()`, `avenrun[]` | `<linux/sched/loadavg.h>` |

### CPU count — `num_online_cpus()`

```c
seq_printf(m, "CPU cores : %d online  (%d possible)\n",
           num_online_cpus(), num_possible_cpus());
```

- `num_online_cpus()` — CPUs currently active on the scheduler's run queues.
  A hotplug event (CPU offline) can lower this count at runtime.
- `num_possible_cpus()` — the hardware maximum detected at boot (from ACPI /
  device tree). This is the ceiling; `online ≤ possible`.

### Kernel version — `utsname()->release`

```c
seq_printf(m, "Kernel version : %s\n", utsname()->release);
```

`utsname()` returns a pointer to the UTS (Unix Time-sharing System) namespace
struct for the current process. The `.release` field contains the same string
printed by `uname -r` — it is set at build time from the kernel `Makefile`
and baked into the image. It is read-only at runtime; safe to access without
any lock.

### Uptime — `jiffies` and `HZ`

```c
jiffies_to_timespec64(jiffies - INITIAL_JIFFIES, &ts);
```

- `jiffies` — a global counter incremented by the timer interrupt exactly
  `HZ` times per second. `HZ` is typically 250 or 1000, depending on
  `CONFIG_HZ`. Never read `jiffies` twice in a single calculation — use
  `READ_ONCE(jiffies)` or call `jiffies_to_timespec64()` which handles it.
- `INITIAL_JIFFIES` — the value `jiffies` had at boot. Subtracting it gives
  elapsed ticks since boot.
- `jiffies_to_timespec64()` — converts the tick count to seconds and
  nanoseconds, correctly handling the 32-bit wraparound that occurs roughly
  every 50 days at `HZ=1000`.

### Load average — `get_avenrun()`

```c
get_avenrun(avnrun, FIXED_1 / 200, 0);
seq_printf(m, "%lu.%02lu", LOAD_INT(avnrun[0]), LOAD_FRAC(avnrun[0]));
```

The kernel stores three exponential moving averages of the run-queue length
in the global array `avenrun[3]` — for 1, 5, and 15 minutes.

Values are **fixed-point integers** scaled by `FIXED_1 = 2048` (i.e., shifted
left by 11 bits). To print them as decimals:

```c
LOAD_INT(x)  = x >> FSHIFT          /* integer part */
LOAD_FRAC(x) = ((x & (FIXED_1-1)) * 100 + FIXED_1/2) / FIXED_1
```

`get_avenrun()` copies the array safely; reading `avenrun[]` directly can
race with `calc_global_load()`, which updates the values every 5 scheduler
ticks.

## procfs

`/proc` is a **pseudo-filesystem** — nothing in it lives on disk. The kernel
mounts it at boot and populates it on demand: a file only "exists" when
something reads it, and its content is generated by whatever kernel function
is registered to handle that path.

### Registration

```c
proc_entry = proc_create("sysinfo_plus", 0444, NULL, &sysinfo_fops);
```

- First argument — the filename under `/proc/`.
- `0444` — permission bits: world-readable, no write.
- `NULL` parent — place directly under `/proc/` (not in a subdirectory).
- `&sysinfo_fops` — our `proc_ops` struct wiring up `open`/`read`/`lseek`/
  `release` to the seq_file helpers.

The returned `proc_dir_entry *` is stored and passed to `proc_remove()` on
unload. Failing to call `proc_remove()` leaves a dangling pointer in the
procfs tree — the entry exists but its backing code is gone, causing a kernel
panic on the next read.

### The seq_file layer

`seq_file` sits between the VFS and our show function. It handles:

- **Buffering** — accumulates `seq_printf()` calls into an internal buffer.
- **Pagination** — if the buffer is larger than one `read()` call's buffer,
  seq_file splits it across multiple reads transparently.
- **Seeking** — `seq_lseek` implements `lseek()` on the virtual file.

`single_open()` is the simplest entry point: it calls our `sysinfo_show()`
exactly once per `open()` and manages the full start/next/stop lifecycle
internally.

### Full call chain

```
cat /proc/sysinfo_plus
  │
  ├─ open()  ──→ VFS ──→ proc_ops.proc_open
  │                           └─ single_open() ──→ sysinfo_open()
  │
  ├─ read()  ──→ VFS ──→ seq_read()
  │                           └─ sysinfo_show()   ← our code runs here
  │                                 ├─ si_meminfo()
  │                                 ├─ num_online_cpus()
  │                                 ├─ utsname()->release
  │                                 ├─ format_uptime()
  │                                 └─ format_loadavg()
  │
  └─ close() ──→ single_release()
```

## Memory Information

Memory data comes from `si_meminfo()`, which fills a `struct sysinfo`:

```c
struct sysinfo {
    long uptime;            /* NOT used — prefer jiffies for uptime */
    unsigned long loads[3]; /* NOT used — prefer get_avenrun() */
    unsigned long totalram; /* total usable RAM, in pages */
    unsigned long freeram;  /* free RAM pages */
    unsigned long sharedram;/* shared memory pages */
    unsigned long bufferram;/* pages used for buffer cache */
    unsigned long totalswap;/* total swap pages */
    unsigned long freeswap; /* free swap pages */
    unsigned short procs;   /* number of current processes */
    unsigned long mem_unit; /* size of a memory unit in bytes */
};
```

> **Important:** All `*ram` fields are in **pages**, not bytes.  
> `mem_unit` equals `PAGE_SIZE` (4096 bytes on x86/ARM).  
> Multiply by `mem_unit` to get bytes, or right-shift by 20 to get MiB:
> ```c
> total_mb = (si.totalram * si.mem_unit) >> 20;
> ```

### Why not read `/proc/meminfo` directly?

Other kernel code (and `/proc/meminfo` itself) calls `si_meminfo()` for the
same data. Reading `/proc/meminfo` from inside the kernel would be circular
and require opening a VFS file from kernel context — fragile and unnecessary.
`si_meminfo()` is the authoritative single source.

### RAM vs MemAvailable

`freeram` corresponds to `/proc/meminfo`'s `MemFree` — pages with no content
at all. The more useful `MemAvailable` (pages reclaimable without swapping)
requires `si_meminfo_node_reclaim()` and page-cache accounting, which is
outside the scope of this module.

## All Fields Explained

| Field | Source | Notes |
|---|---|---|
| **CPU cores online** | `num_online_cpus()` | Can change with CPU hotplug |
| **CPU cores possible** | `num_possible_cpus()` | Hardware maximum, fixed at boot |
| **Kernel version** | `utsname()->release` | Same as `uname -r` |
| **Total RAM** | `si.totalram * si.mem_unit` | Usable RAM (excludes firmware/kernel reserves) |
| **Free RAM** | `si.freeram * si.mem_unit` | Completely unused pages |
| **Used RAM** | `total − free` | Includes file cache; not a measure of "pressure" |
| **Buffer cache** | `si.bufferram * si.mem_unit` | Cached disk block metadata |
| **Uptime** | `jiffies − INITIAL_JIFFIES` | Monotonic; unaffected by NTP or time changes |
| **Load avg 1 min** | `avenrun[0]` | Exponential moving average of run-queue depth |
| **Load avg 5 min** | `avenrun[1]` | Smoothed over 5 minutes |
| **Load avg 15 min** | `avenrun[2]` | Long-term trend indicator |

## Module Walkthrough

### `sysinfo_show()` — the core function

```c
static int sysinfo_show(struct seq_file *m, void *v)
{
    struct sysinfo si;
    si_meminfo(&si);

    unsigned long total_mb = (si.totalram * si.mem_unit) >> 20;
    unsigned long free_mb  = (si.freeram  * si.mem_unit) >> 20;
    unsigned long used_mb  = total_mb - free_mb;

    seq_printf(m, "CPU cores       : %d online  (%d possible)\n",
               num_online_cpus(), num_possible_cpus());

    seq_printf(m, "Kernel version  : %s\n", utsname()->release);

    seq_printf(m, "Total RAM       : %lu MiB\n", total_mb);
    seq_printf(m, "Free RAM        : %lu MiB\n", free_mb);
    seq_printf(m, "Used RAM        : %lu MiB\n", used_mb);

    format_uptime(m);
    format_loadavg(m);
    return 0;
}
```

### `format_uptime()` — jiffies to human time

```c
static void format_uptime(struct seq_file *m)
{
    struct timespec64 ts;
    jiffies_to_timespec64(jiffies - INITIAL_JIFFIES, &ts);

    unsigned long secs  = ts.tv_sec;
    unsigned long days  = secs / 86400; secs %= 86400;
    unsigned long hours = secs / 3600;  secs %= 3600;
    unsigned long mins  = secs / 60;    secs %= 60;

    seq_printf(m, "Uptime : %lu days, %lu hours, %lu min, %lu sec\n",
               days, hours, mins, secs);
}
```

### `format_loadavg()` — fixed-point decoding

```c
static void format_loadavg(struct seq_file *m)
{
    unsigned long avnrun[3];
    get_avenrun(avnrun, FIXED_1 / 200, 0);

    seq_printf(m, "Load average : %lu.%02lu  %lu.%02lu  %lu.%02lu\n",
               LOAD_INT(avnrun[0]), LOAD_FRAC(avnrun[0]),
               LOAD_INT(avnrun[1]), LOAD_FRAC(avnrun[1]),
               LOAD_INT(avnrun[2]), LOAD_FRAC(avnrun[2]));
}
```

### Module init / exit

```c
static int __init sysinfo_init(void)
{
    proc_entry = proc_create("sysinfo_plus", 0444, NULL, &sysinfo_fops);
    if (!proc_entry) return -ENOMEM;
    return 0;
}

static void __exit sysinfo_exit(void)
{
    proc_remove(proc_entry);   /* always pass the stored pointer */
}
```

> **Store the `proc_dir_entry *`.**  
> Passing `NULL` to `proc_remove()` on some kernel versions removes nothing
> silently. Always store the pointer returned by `proc_create()` and pass it
> to `proc_remove()` on exit.

## Compatibility

| Kernel range | Status | Notes |
|---|---|---|
| ≥ 5.6 | Supported | Uses `proc_ops` (replaced `file_operations` for procfs) |
| 4.x – 5.5 | Minor change | Replace `proc_ops` with `file_operations`; use `.open`/`.read` keys |
| < 4.x | Not tested | `get_avenrun()` signature may differ |
